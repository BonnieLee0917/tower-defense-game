## 一、产品概述

### 1.1 一句话描述
一款浏览器端的塔防策略游戏，玩家通过建造和升级防御塔，阻止敌人到达终点。

### 1.2 目标用户
- 休闲游戏玩家
- 喜欢策略/塔防类游戏的玩家
- PC/平板浏览器用户

### 1.3 核心体验
**"用有限资源做最优防御决策"** — 在哪建塔、建什么塔、什么时候升级，每个选择都影响战局。

## 二、MVP 范围（第一版）

### 2.1 包含 ✅
| 模块 | 内容 |
|------|------|
| 地图 | 1 张完整地图，含固定敌人路径和 8-12 个建塔点 |
| 防御塔 | 4 种基础塔（弓箭塔、魔法塔、炮塔、兵营） |
| 升级 | 每种塔 3 级，升级提升伤害/范围/特效 |
| 敌人 | 4 种敌人类型（普通兵、快速兵、重甲兵、飞行单位） |
| 波次 | 10 波敌人，递增难度 |
| 经济 | 初始金币 + 击杀奖励，用于建塔和升级 |
| 生命值 | 玩家有 20 点生命值，敌人到达终点扣除 |
| UI | 开始界面、游戏 HUD、胜利/失败画面 |
| 操控 | 鼠标点击建塔/升级/出售，支持暂停和加速 |

### 2.2 不包含 ❌（后续版本）
- 英雄系统
- 多张地图 / 关卡选择
- 技能树 / 科技升级
- 存档系统
- 排行榜 / 多人
- 音效和背景音乐（MVP 可选加入）
- 移动端适配

## 三、核心玩法设计

### 3.1 游戏流程
```
开始界面 → 进入关卡 → 准备阶段（建塔）→ 波次开始 → 
击败所有波次 → 胜利 / 生命值归零 → 失败 → 重试
```

### 3.2 防御塔设计

| 塔类型 | 攻击方式 | 特点 | 价格 |
|--------|---------|------|------|
| 🏹 弓箭塔 | 单体远程 | 攻速快，伤害低，性价比高 | 70 金 |
| 🔮 魔法塔 | 范围魔法 | 攻速慢，伤害高，可减速 | 100 金 |
| 💣 炮塔 | 范围物理 | 溅射伤害，对群体有效 | 125 金 |
| ⚔️ 兵营 | 近战拦截 | 派出士兵阻挡敌人前进 | 70 金 |

**升级费用：**
- 1→2 级：基础价格 × 1.0
- 2→3 级：基础价格 × 1.5
- 出售返还：总投入的 60%

### 3.3 敌人设计

| 敌人类型 | 血量 | 速度 | 护甲 | 特殊 | 击杀奖励 |
|---------|------|------|------|------|---------|
| 🧟 普通兵 | 低 | 中 | 无 | 无 | 5 金 |
| 🐺 快速兵 | 低 | 快 | 无 | 难以被兵营拦截 | 7 金 |
| 🛡️ 重甲兵 | 高 | 慢 | 高物理护甲 | 魔法伤害更有效 | 15 金 |
| 🦅 飞行单位 | 中 | 中 | 无 | 无视兵营，只能被远程塔攻击 | 10 金 |

### 3.4 波次设计（10 波）

| 波次 | 敌人组成 | 数量 | 难度 |
|------|---------|------|------|
| 1 | 普通兵 | 8 | ⭐ |
| 2 | 普通兵 + 快速兵 | 10 | ⭐ |
| 3 | 普通兵 + 重甲兵 | 10 | ⭐⭐ |
| 4 | 快速兵群 | 15 | ⭐⭐ |
| 5 | 混合（普通+快速+重甲） | 12 | ⭐⭐⭐ |
| 6 | 飞行单位首次出现 | 10 | ⭐⭐⭐ |
| 7 | 重甲兵 + 飞行单位 | 12 | ⭐⭐⭐ |
| 8 | 全类型混合 | 15 | ⭐⭐⭐⭐ |
| 9 | 快速兵 + 飞行大群 | 18 | ⭐⭐⭐⭐ |
| 10 | 全类型 + BOSS（高血量重甲） | 15+1 | ⭐⭐⭐⭐⭐ |

### 3.5 经济系统
- **初始金币：** 200
- **击杀奖励：** 见敌人表
- **提前发波奖励：** 当前波未结束时手动触发下一波，+20 金奖励
- **通关星级：** 0 漏 = 3⭐ / 漏 1-5 = 2⭐ / 漏 6-19 = 1⭐

## 四、UI/交互设计

### 4.1 开始界面
- 游戏 Logo + 标题
- 「开始游戏」按钮
- 简单的操作说明

### 4.2 游戏 HUD
```
┌─────────────────────────────────┐
│  ❤️ 20/20    💰 200    🌊 1/10  │  ← 顶部状态栏
│                                 │
│         [ 游 戏 地 图 ]          │
│                                 │
│  ⏸️ 暂停   ⏩ 加速   📢 发波    │  ← 底部控制栏
└─────────────────────────────────┘
```

### 4.3 建塔交互
1. 点击空建塔点 → 弹出塔选择环形菜单（4 个方向对应 4 种塔）
2. 点击已有塔 → 弹出升级/出售菜单
3. 金币不足时，对应选项置灰

### 4.4 信息展示
- 塔的攻击范围：选中时显示半透明圆圈
- 敌人血条：显示在敌人头顶
- 伤害数字：可选的飘字效果

## 五、美术风格

### 建议方向（待确认）
| 风格 | 优势 | 劣势 |
|------|------|------|
| 🎨 卡通手绘（类 KR） | 亲切感强，辨识度高 | 美术成本高 |
| 👾 像素风 | 开发快，素材易找 | 可能显得简陋 |
| 🧊 Low-poly / 扁平化 | 现代感，容易做得好看 | 跟 KR 差异大 |

**推荐：扁平卡通风** — 兼顾美观和开发效率，可用 AI 生成辅助。

## 六、技术方案

### 6.1 技术栈
- **引擎：** Phaser 3（成熟的 2D Web 游戏框架）
- **语言：** TypeScript
- **构建：** Vite
- **部署：** Vercel 或 Netlify（免费）

### 6.2 核心架构
```
src/
├── scenes/          # 场景（菜单、游戏、结算）
├── entities/
│   ├── towers/      # 塔类
│   ├── enemies/     # 敌人类
│   └── projectiles/ # 弹道
├── systems/
│   ├── WaveManager  # 波次管理
│   ├── EconomyManager # 经济系统
│   └── PathManager  # 路径系统
├── ui/              # UI 组件
├── maps/            # 地图数据（JSON）
├── config/          # 平衡性配置
└── assets/          # 图片、动画
```

### 6.3 地图数据格式
地图用 JSON 定义，方便后续扩展：
```json
{
  "name": "绿野山谷",
  "path": [[0,300], [200,300], [200,100], [500,100], [500,400], [800,400]],
  "buildSpots": [
    {"x": 150, "y": 250, "id": 1},
    {"x": 350, "y": 150, "id": 2}
  ],
  "waves": "waves_level1.json"
}
```

## 八、后续扩展路线

完成 MVP 后可以逐步加入：

1. **更多关卡** — 3-5 张不同地图，多条路径
2. **英雄系统** — 可操控的英雄单位 + 技能
3. **塔的高级分支** — 3 级后分化成 2 个特化方向
4. **特殊技能** — 全屏轰炸、援军召唤等主动技能
5. **难度模式** — 普通/困难/噩梦
6. **存档系统** — LocalStorage 或云存档
7. **排行榜** — 通关时间/星级排名
8. **音效音乐** — BGM + 攻击/建造/死亡音效
9. **移动端适配** — 触屏操作
10. **关卡编辑器** — 让玩家自创关卡
