> 项目：Tower Storm
> 作者：Bonnie PM
> 日期：2026-04-07
> 目的：为 Tower Storm 的 Phase 2 产品方向、玩法策略、经济平衡和 PRD 修订提供依据

## 1. 结论先行（Executive Summary）

### 1.1 对 Tower Storm 最重要的 8 条结论

1. **Kingdom Rush 是经典塔防模式，不是每波重置模式。**
   - 同一关卡内，**塔 / 金币 / 地图状态跨 wave 保留**。
   - 只有切换到新关卡时才重置。

2. **第一波前有准备阶段，后续波次自动推进。**
   - 第一波前玩家先用启动资金建塔。
   - 后续 wave 之间有 **自动倒计时**。
   - 玩家可以**提前发波**跳过倒计时，换取额外收益。

3. **Kingdom Rush 的策略深度来自“跨波次积累与升级”。**
   - 不是每波解一道独立题。
   - 玩家核心决策是：哪里建、什么时候升、什么时候卖、什么时候提前发波。

4. **经济系统必须允许“开局先布一个基本防线”。**
   - 启动资金通常足够放下 **3-4 个低阶决策点**，而不是只能勉强摆 1-2 座塔。
   - 我们把启动资金从 200 调到 **350** 是正确方向。

5. **难度曲线必须前低后高。**
   - 前 1-2 波用于教学和建立节奏。
   - 中期开始引入混合敌人与克制关系。
   - 后期才给玩家明显压迫感。

6. **Kingdom Rush 的核心塔体系不是“很多塔”，而是“少量基础塔 + 明确克制 + 升级分支”。**
   - 4 个基础塔型就足够撑起完整策略。
   - 真正的深度来自升级与组合，而不是初始塔数量。

7. **UI/交互是策略表达的一部分。**
   - KR 的环形建塔菜单、清晰 HUD、提前发波按钮、本质上都在降低操作成本，放大策略决策。

8. **Tower Storm Phase 2 应优先补“策略完整性”，不是盲目加内容。**
   - 优先级应是：
     1) 4 塔 + 4 敌人 + 克制关系
     2) 升级/出售
     3) 完整波次节奏
     4) 数值平衡
     5) 基础移动兼容

## 2. Kingdom Rush 的核心玩法循环

### 2.1 单关核心循环

```text
进入关卡
→ 观察地图与建塔点
→ 用启动资金布初始防线
→ 手动开始第一波
→ 敌人来袭
→ 击杀敌人获得金币
→ 波间自动倒计时
→ 玩家在倒计时中补塔 / 升级 / 选择是否提前发波
→ 后续 wave 难度递增
→ 打完整关
→ 根据剩余生命评星
→ 获得星星用于全局升级
```

### 2.2 真正的核心体验

Kingdom Rush 的核心不是“摆塔”本身，而是：

**用有限资源在有限时间内，持续构建一套越来越强的防御体系。**

也就是说，策略张力来自：
- 当前金币不够，先补新塔还是升旧塔？
- 当前波很稳，要不要提前发波赚额外收益？
- 面对高护甲敌人，是补法师塔，还是让兵营先拖住？
- 现有塔位布局是否应该卖掉重建？

## 3. 关卡结构（Level / Wave / 进度系统）

### 3.1 Level 与 Wave 的关系

KR 里有两个层级：

- **Level / Stage（关卡）**：一整张地图，一整轮完整挑战
- **Wave（波次）**：关卡中的敌人批次

**关键点：wave 不是独立关卡。**

在一个 level 中：
- 地图不变
- 建塔点不变
- 已建塔保留
- 金币累计
- 生命连续继承

### 3.2 关卡数量与波次数

基于 KR 系列常见结构：
- Campaign 一般是 **10+ 主关卡** 串在世界地图上
- 每关常见 **约 10-15 波**
- 新手关前几波用于教学
- 后期关卡引入更复杂的敌人组合、分路、BOSS 或特殊机制

### 3.3 星级 / 进度

KR 常见评星逻辑：
- 20 条命起步
- 结算按剩余命数给 1-3 星
- 星星用于解锁全局升级

对 Tower Storm 的启发：
- 我们后续应补 **三星评价**，而不仅是赢/输
- 这会增强复玩价值与“更优解”动机

## 4. 波次节奏设计

### 4.1 标准节奏

KR 更接近下面这个节奏：

- **第一波前：准备阶段**
  - 玩家建初始塔
  - 手动点击开始

- **后续波之间：自动倒计时**
  - 给玩家 10-15s 左右调整布防
  - 倒计时结束自动开下一波

- **玩家可提前发波**
  - 提前跳过等待
  - 获得额外金币/额外收益
  - 本质是一个风险收益选择

### 4.2 对 Tower Storm 的落地建议

当前我们已锁定的方向是正确的：
- 第一波手动开始
- 后续 15s 自动倒计时
- 倒计时期间允许提前发波 + 奖励金币

### 4.3 MVP / Phase 2 波次建议

#### MVP（当前）
- 5 波保留
- 第 1 波极弱，作为教学波
- 第 2-3 波建立节奏
- 第 4-5 波给第一次真正压力

#### Phase 2/3
- 扩到 **10-12 波**
- 节奏建议：
  - Wave 1-2：教学 / 建立布局
  - Wave 3-5：加入快速兵 / 高血量兵
  - Wave 6-8：混合克制开始成型
  - Wave 9-10：高压混合波
  - Wave 10+/Boss：收束关卡情绪

## 5. 经济系统调研与建议

### 5.1 KR 经济的本质

KR 的经济不是“越多越好”，而是：
- 开局给足够资源完成基本布防
- 中后期让击杀奖励驱动玩家持续扩张与升级
- 通过提前发波、卖塔、升级等动作制造资源张力

### 5.2 启动资金经验结论

调研结论：
- KR 新手关启动资金通常会让玩家**能布出一套最低限度有效防线**
- 不会让玩家只够放一两座塔就被迫开波

结合我们当前价格：
- 弓箭塔 70
- 炮塔 125

如果启动资金只有 **200**：
- 只能做 2 座左右塔的决策
- 玩家容错太低
- 开局布局空间太小

如果启动资金 **350**：
- 可支持 3-4 座基础防御
- 开局更接近 KR 的“先布基本盘，再边打边扩”节奏

### 5.3 当前推荐 baseline

**Tower Storm 当前 baseline：350 金**

理由：
- 允许开局完成基本布防
- 不会让第一波过于“裸奔”
- 更适合新手理解建塔节奏

### 5.4 后续经济调优框架

后续经济建议按这 4 个变量联动调：
1. 启动资金
2. 塔基础价格
3. 击杀奖励
4. 提前发波奖励

建议原则：
- 玩家开局应能摆 **3-4 个低阶决策点**
- 普通玩家在前 2-3 波后，应该能自然做出 **第一次升级 or 第一次塔型扩展**
- 提前发波奖励应“有诱惑但不破坏平衡”

## 6. 塔体系设计（KR 的策略骨架）

### 6.1 KR 的基础塔型

KR 的经典 4 塔框架是：

1. **Archer Tower（弓箭塔）**
   - 对空友好
   - 攻速快
   - 单体 DPS 稳定
   - 泛用型、高性价比

2. **Mage Tower（法师塔）**
   - 打魔法伤害
   - 克制高护甲敌人
   - 往往有减速/控制/特殊能力

3. **Artillery（炮塔）**
   - 群伤 / AoE
   - 擅长清杂兵
   - 对付成群敌人效率高

4. **Barracks（兵营）**
   - 产出地面单位阻挡敌人
   - 提供“停留时间”，让其他塔多打几轮
   - 是策略中的“控制位”而不是纯输出位

### 6.2 KR 的深度来源：升级分支

KR 的深度并不依赖很多基础塔型，而是：
- 基础塔先升到中阶
- 再分化成两个高级专精方向

这意味着：
- 玩家前期好理解（只有 4 类）
- 中后期仍有策略深度（专精分叉）

### 6.3 对 Tower Storm 的建议

#### Phase 2 必做
- 补齐 4 塔：弓箭 / 法师 / 炮塔 / 兵营
- 每种塔先做 3 级基础升级
- 先不急着做 Tier 4 分叉，但代码结构要预留

#### Phase 3/4 才考虑
- 高级专精分支
- 特殊技能
- 更复杂的技能树

## 7. 敌人体系与克制关系

### 7.1 KR 敌人的设计原则

KR 敌人不是简单堆数值，而是围绕几种关键属性展开：
- **血量（HP）**
- **移动速度**
- **护甲（Armor）**
- **魔抗（Magic Resistance）**
- **飞行 / 地面**
- **是否召唤 / 是否再生 / 是否特殊技能**

### 7.2 典型敌人分层

Tower Storm 后续可参照 KR 的方法，将敌人按功能角色设计：

1. **普通兵**
   - 新手教学敌人
   - 无特殊属性

2. **快速兵**
   - 压迫防线节奏
   - 克制低攻速塔布局

3. **重甲兵**
   - 迫使玩家补法师塔
   - 建立“物理 vs 魔法”克制关系

4. **飞行敌人**
   - 强迫玩家关注对空能力
   - 避免只靠地面拦截解决一切

5. **精英 / BOSS**
   - 提供关卡高潮
   - 测试玩家整体布防是否合理

### 7.3 Tower Storm 当前建议

#### Phase 2
- 普通兵
- 快速兵
- 重甲兵
- 飞行敌人

#### Phase 3
- 混合精英敌人
- BOSS
- 更复杂被动（召唤、护盾、分裂、回血）

## 8. 升级、出售与资源回收

### 8.1 升级

KR 的核心策略之一就是：
- 不是不断新建低级塔
- 而是在关键 chokepoint 处持续升级核心塔位

启发：
- Tower Storm 必须尽快补 **升级系统**
- 没有升级，策略深度会明显不够

### 8.2 出售

KR 常见机制：
- 卖塔返还约 **60%** 投入

这很关键，因为它允许玩家：
- 前期开局先用临时塔过渡
- 中后期卖掉重构布局
- 尝试不同策略时不至于完全崩盘

**建议：Tower Storm 按 60% 返还设计。**

## 9. 主动技能与额外策略层

### 9.1 KR 的常见主动技能

KR 系列的额外策略层通常包括：
- **Rain of Fire**：范围爆发救场
- **Reinforcements**：临时援军拖时间

这些不是主塔体系，但能显著提升策略表达和救场空间。

### 9.2 对 Tower Storm 的建议

- MVP / Phase 2：**先不做**，避免系统过多
- Phase 3 之后再补主动技能

原因：
- 现在真正缺的是塔体系完整性与升级深度
- 不是额外按钮数量

## 10. 英雄系统

### 10.1 KR 系列中的位置

在 KR 系列中，英雄系统是重要策略层，尤其在后续作品中更明显：
- 英雄可移动
- 有主动/被动技能
- 承担补位、救场、局部爆发等职责

### 10.2 对 Tower Storm 的建议

- **MVP / Phase 2 不做英雄**
- 先把塔防核心循环做扎实
- 英雄应放在后续版本，而不是当前阶段抢优先级

## 11. UI / 交互策略启发

虽然详细设计由 Vivian 主导，但产品上需要明确 KR 的几个关键交互原则：

1. **建塔动作必须快**
   - 点击塔位 → 快速选塔
   - 环形菜单优于长列表

2. **HUD 要清晰但不压地图**
   - 关键资源（命 / 金 / 波次 / 倒计时）一眼可读

3. **提前发波必须是显式风险收益按钮**
   - 玩家应明确知道：我现在可以冒险换奖励

4. **反馈要直观**
   - 命中、掉血、飘字、建塔、升级、发波都应有明显反馈

## 12. 对 Tower Storm 的产品修订建议

### 12.1 当前不变的 5 点（已锁定）

1. **KR 经典模式**：塔 / 金币 / 地图跨 wave 保留
2. **第一波手动开始，后续自动倒计时**
3. **启动资金 350 作为 baseline**
4. **难度曲线前低后高**
5. **移动端优先级：Phase 2 基础兼容，Phase 4 完整适配**

### 12.2 Phase 2 推荐范围

#### 功能
- 4 塔补齐：弓箭 / 法师 / 炮塔 / 兵营
- 4 敌人补齐：普通 / 快速 / 重甲 / 飞行
- 升级系统（至少 3 级）
- 出售系统（60%）
- 波次节奏优化（10-12 波）
- 更平滑难度曲线

#### 交互
- 环形建塔菜单
- 更明确的波次倒计时反馈
- 更好的建塔 / 升级 / 出售操作流

#### 技术/兼容
- 基础移动兼容（横屏优先、画面不变形、主要 HUD 可用）

### 12.3 Phase 3/4 再做

- 高级专精分支
- BOSS
- 主动技能
- 英雄
- 完整移动端适配
- 更精致美术与特效

## 14. 最终建议（PM 结论）

如果只用一句话概括：

**Tower Storm 下一阶段最该做的，不是“加更多东西”，而是把 Kingdom Rush 那套经典塔防的策略骨架补完整。**

也就是：
- 更清晰的波次节奏
- 更合理的开局经济
- 更完整的塔/敌人克制
- 更有深度的升级/出售系统
- 更高效的建塔交互

只要这些补齐，产品策略就会从“一个能玩的原型”，升级成“一个真正有塔防味道的游戏”。
