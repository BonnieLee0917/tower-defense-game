# Tower Storm — 视觉大改实施方案

> 作者：Vivian（Designer）
> 日期：2026-04-08
> 目标：从当前 Graphics API placeholder 版本，升级到更接近 Kingdom Rush 的完整视觉形态

---

## 1. 结论先行

### 推荐执行路线
我建议采用 **两阶段视觉升级**，而不是一次性重做全部：

### 方案 A — 像素风快速成型（推荐作为当前执行方案）
- **目标：1-2 天内让游戏从“占位图”升级到“完整有皮肤版本”**
- 方式：使用统一生态的免费像素风素材包 + 少量 AI / 后处理补缺
- 优点：快、统一、开发集成成本低、能立刻提升体验
- 缺点：和 KR 的高清卡通手绘风仍有距离

### 方案 B — 卡通手绘风最终目标（中期方案）
- **目标：1-2 周内逐步逼近 KR 的最终质感**
- 方式：AI 生成 + 手动筛选 + 统一后处理 + 局部定制
- 优点：更贴近 KR
- 缺点：素材不稳定、统一成本高、周期长

### 我的建议
**先落地方案 A，再以方案 B 作为后续迭代方向。**

原因：
1. 小萌明确不想再看 placeholder
2. 当前瓶颈在于“没有素材”，不是“设计方向不清楚”
3. 像素风虽然不完全像 KR，但能最快把产品从 demo 推到可交付状态
4. 后续如果小萌坚持 KR 手绘风，我们再替换素材层，不必重写玩法逻辑

## 2. 当前视觉短板 Top 3（优先级）

### P0. 地图还是纯色块
**问题：**
- 草地、路径都是纯色格子
- 没有地面纹理、边缘变化、环境装饰
- 第一眼就暴露出 prototype 感

**改法：**
- 上 tileset
- 草地 / 路径 / 建塔点 / 障碍物分层
- 增加树、石头、旗帜、栅栏等环境装饰

**价值：** 最大，投入产出比最高

---

### P1. 塔和敌人还是几何图形
**问题：**
- 即使形状语言已建立，本质上仍是图形原型
- 游戏感、角色感、世界观都不足

**改法：**
- 塔和敌人换成 sprite / spritesheet
- 至少具备 idle / attack / death / move 的基础状态

**价值：** 高，但依赖素材到位

---

### P2. UI 包装感不足
**问题：**
- HUD 还是文字 + emoji
- 按钮、面板、结算画面缺少成品感
- 环形菜单交互已经对了，但视觉还不够“像 KR”

**改法：**
- HUD 图标化
- 按钮加质感
- 面板和结算页做视觉包装

**价值：** 中高，成本相对低，见效快

## 4. 地图升级方案（12:30 先交的部分）

## 4.1 目标
让地图从纯色块变成“有世界感的可玩关卡”。

## 4.2 实施路径
### Step 1：最小替换
- 草地 tile 替换纯色格
- 路径 tile 替换棕色矩形
- 建塔点保留当前逻辑，但增加底板/标记素材

### Step 2：加装饰
- 树木、石头、栅栏、旗帜、花草
- 不影响路径和建塔点逻辑

### Step 3：切换 Tiled
- 从 `drawMap()` 手工绘制，切到 tilemap JSON
- 路径、装饰、建塔点分层管理

## 4.3 尺寸规范
- 素材原始：32×32
- 游戏基准：64×64
- 渲染方式：2x 最近邻缩放（NEAREST）

## 4.4 Haaland 开发工作量预估
- **Step 1（直接 image 替换）：2-3h**
- **Step 2（装饰层）：1-2h**
- **Step 3（Tiled 接入）：3-5h**

## 5. 塔 & 敌人 sprite 替换方案

## 5.1 塔
### 第一批必须替换
- 弓箭塔
- 魔法塔
- 炮塔
- 兵营

### 每个塔至少需要
- idle 1 帧（最低）
- attack 1-2 帧（可选）
- upgrade 通过换 sprite 或覆盖层表示

### 推荐做法
- 第一版：先只替换静态 sprite
- 第二版：再补 attack 动画

## 5.2 敌人
### 第一批必须替换
- 普通兵
- 快速兵
- 重甲兵
- 飞行单位

### 每个敌人至少需要
- move 动画 / 2-4 帧
- death 动画 / 1-2 帧（最低）
- 飞行单位额外需要翅膀动作

## 5.3 Haaland 开发工作量预估
- **静态 sprite 替换：3-4h**
- **基础 move / death 动画：2-4h**
- **atlas / spritesheet 管理：1-2h**

## 7. AI 补缺策略

如果走像素风主路线，AI 不应该生成整套地图，而应用在这些地方：
- 缺失塔类型的变体
- 特定 Boss / 精英怪
- UI 插图 / 背景插画
- 特殊装饰物

如果走卡通手绘风路线，AI 需要生成：
- 地图 tile 纹理
- 4 塔 4 敌人基础 sprite
- HUD 图标
- 结算页装饰

**风险：** 风格不稳，需要统一后处理

## 8. 素材 pipeline

### 推荐 pipeline
1. **筛素材包**（我完成）
2. **下载并归类**
3. **统一导出规范**
   - 文件名
   - 尺寸
   - 放大倍数
   - 透明背景
4. **必要的 AI 补缺**
5. **后处理统一风格**
6. **交给 Haaland 集成**

### 目录建议
- `assets/tiles/`
- `assets/towers/`
- `assets/enemies/`
- `assets/ui/`
- `assets/effects/`
- `assets/source/`（原始下载包）
